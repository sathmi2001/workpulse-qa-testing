# Test Cases

## Overview

This folder contains the manual functional test cases created for the WorkPulse Desktop Application.

The test cases were designed to validate the application's core business functionality and ensure that user requirements were correctly implemented.

## Modules Covered

### User Management

* Home Page Navigation
* Employee Registration
* Login Validation
* Input Validation
* Form Validation
* User Authentication

### Leave Management

* Leave Request Submission
* Leave Form Validation
* Leave Approval Workflow
* Invalid Input Validation
* Required Field Validation

## Test Design Techniques

* Functional Testing
* Positive Testing
* Negative Testing
* Boundary Value Validation
* Input Validation
* Regression Testing

## Test Case Structure

Each test case includes:

* Test Case ID
* Test Scenario
* Test Case Description
* Preconditions
* Test Steps
* Test Data
* Expected Result
* Actual Result
* Test Status
* Priority
* QA Comments

These test cases were executed to verify application behavior and identify functional defects before deployment.
